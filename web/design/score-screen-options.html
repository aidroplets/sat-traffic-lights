Score-screen redesign · 5 options

Same content (subjects + difficulty + target score + start), five different ways to compose it. Mockups are static — no JS, just look-and-feel. Each option uses the same visual language as the live app so we're comparing structure, not styling.

Option A

Compact — segmented difficulty

Same vertical order as today, but tighter. The four difficulty tiles collapse to a single segmented control (iOS-style), halving the difficulty surface. Target score is a small inline number, not a full input row.

Conversational — sentence with editable tokens

Option F · Hybrid

Hybrid — keep tiles, drop the input

Closest to the current screen, but removes the redundant Target Score input. The selected tier IS your target — picking Medium auto-sets target to the band's top. Power users who want a precise score click "Set custom" to reveal the input on demand.

Tradeoffs at a glance

OptionBest forProsCons
A · Compact Mobile-first; users who already know what they want Halves vertical space. Segmented control reads as one decision. Familiar iOS pattern. Smaller tap targets on the segments. Loses the visual weight of the difficulty tiles.
B · Two-column Desktop; users who edit decisions independently Both decisions visible at once. Cleaner separation. Bottom row pulls toggle + score together. Falls back to stacked on narrow viewports (could become A's worse cousin). More chrome to render.
C · Slider-first New users who think in terms of goal scores, not tiers One control replaces three. Live tier label teaches the mapping. Score is the lead, not a footnote. Less precise (sliders are fiddly at 10-point steps). Subjects feel demoted to bottom.
D · Wizard First-timers; brand-new users who'd freeze at the full form Honors the "Step 1 of 3" copy. Each step is short. Can show pool size + estimate per step. Three clicks to start vs one. Returning users will hate it. Hard to back-edit without unwinding.
E · Conversational Brand-distinctive; works as a marketing differentiator Reads naturally. Memorable. Summary card answers "what am I about to do?" before clicking start. Token pickers are custom UI to build + maintain. Awkward on phones. Could feel cute-not-functional.
F · Hybrid Lowest-risk evolution of the current screen Smallest delta from today. Removes the only redundancy (tier vs target). Retains the four-tile grid users have learned. Doesn't address the longer screen issue. Toggle wording still abstract.
